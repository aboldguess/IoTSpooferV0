IoT Dashboard Spoofer

Emulate multiple devices, spoof identities, and POST telemetry to your dashboard under test.

Quick Start (on-screen instructions)

  1. Create one or more virtual devices with spoofed metadata.
  2. Select a device, optionally attach a webcam frame for camera devices.
  3. Set your dashboard ingest endpoint and authorization header.
  4. Click Generate & Send to emit test payloads.

Dashboard config notes for this spoofer

This spoofer sends JSON telemetry, so configure your dashboard ingestion service to also accept JSON POST requests at the same URL (or route JSON traffic to an equivalent endpoint). Use the Authorization header value below.

  • Endpoint URL: http://192.168.56.1:8001/api/sensor/publish
  • Authorization header: Bearer aYL5A00u1oFbOmmEyu2ZO262krDUNW0jBw0ApS53Dhc
  • HTTP method: POST
  • Content-Type: application/json (automatically set by this app)

If your dashboard requires a custom auth scheme, paste the full header value exactly as expected (for example Token abc123).
